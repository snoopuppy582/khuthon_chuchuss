# 1. 발표 원칙

발표는 어렵게 말하지 않는다. 아래 세 단어만 반복한다.

```
단절 → 이어짐 → 참여
```

> "전통문화의 문제는 사라졌다는 것이 아니라, 현대인의 일상에서 다시 만들어지지 않고 있다는 것입니다."

> "이어줄게는 이 단절을 3D 체험, AI 창작, 실제 방문 연결로 잇습니다."

> "핵심 차별점은 참여입니다."

# 2. 발표 총 구성: 7분

```
[발표 4분] + [시연 3분] = 총 7분
```

발표자는 문제를 길게 설명하지 않는다. 시연으로 빨리 넘어가고, 기능 흐름을 명확히 보여준다.

```

# 4. 시연 파트 (3분) — 상세 시나리오

## 0:00~0:15 — 랜딩 페이지

```
화면: "이어줄게" 랜딩 페이지
동작: [체험 시작하기] 버튼 클릭
멘트: "지금부터 이어줄게의 3D 전통문화 공간에 들어가 보겠습니다."
```

## 0:15~0:45 — 3D 공간 탐색

```
화면: 월정교 교각 → 별자리 파티클 하늘 → 첨성대 등장
동작: 카메라가 천천히 이동하며 공간 전체를 보여줌
멘트: "웹 브라우저에서 누구나 접근할 수 있는 3D 전통문화 공간입니다.
      중앙에는 첨성대가 있고, 주변에는 별자리 효과와 입장 동선이 배치되어 있습니다."
```

## 0:45~1:00 — 첨성대 클릭 → 설명

```
화면: 첨성대 클릭 → 사이드 패널에 "선덕여왕의 첨성대" 설명 표시
동작: 첨성대 오브젝트 클릭
멘트: "첨성대를 클릭하면 문화재 설명과 함께 AI 창작 기능이 열립니다."
```

## 1:00~1:45 — AI 콘텐츠 창작

```
[이미지 — 실시간 생성] (1:00~1:25)
화면: 스타일 "별빛 국악" 선택 → 형식 "이미지" 선택 → [창작하기] 클릭
동작: OpenAI API 호출 → 로딩 애니메이션 → 이미지 생성 완료
멘트: "AI가 첨성대의 별빛 국악 스타일 이미지를 생성하고 있습니다.
      실제로 OpenAI API가 실시간으로 동작합니다."

[노래 — 사전 제작] (1:25~1:35)
화면: 형식 "노래" 선택 → 재생
동작: 사전 제작된 Suno AI 노래 5~8초 재생
멘트: "노래는 시연 안정성을 위해 사전 제작한 결과물을 연결했습니다."

[영상 — 사전 제작] (1:35~1:45)
화면: 형식 "영상" 선택 → 재생
동작: 사전 제작 영상 5~8초 재생
멘트: "영상도 첨성대와 선덕여왕을 주제로 사전 제작해 연결했습니다."
```

## 1:45~2:15 — AI 콘텐츠 게시판

```
화면: 게시판으로 이동 → 방금 만든 이미지 + 다른 사용자 콘텐츠 표시
동작: 게시판의 병풍형 액자들을 보여주고, 카드 하나를 클릭
멘트: "만들어진 콘텐츠는 게시판에 공유됩니다.
      핵심은 전통문화를 관람에서 끝내지 않고, 참여로 이어주는 것입니다."
```

## 2:15~2:45 — QR 오프라인 연결

```
화면: QR 패널 클릭 → QR 코드 표시
동작: QR 코드가 화면에 크게 표시됨
멘트: "마지막으로 이 QR을 스캔하면 첨성대 방문 정보와 주변 코스를 확인할 수 있습니다.
      디지털 체험이 실제 방문으로 이어지는 구조입니다."

## 2:45~3:00 — 시연 마무리

# 5. 심사위원 예상 질문 대응

## Q1. 3D Gaussian Splatting은 구현했나요?

> "이번 MVP에서는 구현하지 않았습니다. 시연 안정성을 위해 Three.js와 공개 3D 에셋으로 구현했습니다. 3DGS는 공개 에셋이 없는 문화유산으로 확장할 때 사용할 수 있는 다음 단계입니다."

## Q2. AI 이미지·노래·영상이 전부 실시간인가요?

> "아닙니다. 이미지는 OpenAI API로 실시간 생성하고, 노래와 영상은 시연 안정성을 위해 사전 제작 결과물을 연결했습니다."

## Q3. 오프라인 연결이 왜 필요한가요?

> "디지털 체험만으로 끝나면 또 하나의 온라인 콘텐츠가 됩니다. 저희는 관심이 실제 문화유산 방문으로 이어지는 구조를 만들고자 했습니다."

## Q4. 부적절한 콘텐츠는 어떻게 제어하나요?

> "MVP에서는 간단한 금칙어 필터와 생성형 AI API의 기본 정책에 의존합니다. 서비스 확장 시에는 별도의 프롬프트 필터링과 게시판 검수 구조를 추가할 수 있습니다."

## Q5. 첨성대 말고 다른 문화재에도 적용 가능한가요?

> "네. 구조는 특정 문화재에 종속되지 않습니다. 문화유산별 3D 공간, 이야기, AI 창작 소재만 바꾸면 다른 문화유산에도 적용할 수 있습니다."

## Q6. 기존 문화유산 정보 서비스와 뭐가 다른가요?

> "핵심 차이는 참여입니다. 기존 서비스가 정보를 보여주는 데 집중했다면, 이어줄게는 사용자가 직접 만들어보고 실제 방문까지 이어지게 합니다."

# 6. 발표 시 주의사항

```
1. 문제를 많이 늘리지 않는다.
   → 루트 원인: 전통문화가 현대인의 일상에서 접근되지도 재생산되지도 않고 있다

2. "정밀 복원"이라고 말하지 않는다.
   → "공개 3D 에셋 기반 체험 공간"

3. "AI가 전부 실시간"이라고 말하지 않는다.
   → "이미지는 실시간, 노래/영상은 사전 제작"

4. 차별점은 하나만 말한다.
   → "참여"

5. 시연 중 API 호출이 실패할 경우
   → "네트워크 상황에 따라 사전 생성 결과를 보여드리겠습니다"
```
