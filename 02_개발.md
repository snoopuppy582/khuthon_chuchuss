
# 2. 3D 에셋 목록

## 메인 에셋: 첨성대

- **출처**: Sketchfab (lacomi1975 제작)
- **스펙**: ~1.1M triangles, ~570k vertices (다운로드 후 최종 확인 필요)
- **라이선스**: CC Attribution 여부 발표 전 최종 확인
- **포맷**: glTF/GLB 다운로드 가능
- **용도**: 3D 공간의 중앙 핵심 오브젝트

## 보조 에셋: 월정교 교각

- **출처**: KHS_Asset (KOREA HERITAGE SERVICE) Sketchfab
- **모델명**: Woljeonggyo Pier, Chunyanggyo & Woljeonggyo
- **스펙**: 61.6k triangles, 31.2k vertices (상대적으로 가벼움)
- **라이선스**: CC Attribution (무료)
- **포맷**: glTF/GLB 다운로드 가능
- **용도**: 입장 동선 — "다리를 건너 첨성대로 향한다"는 시각적 흐름

## Three.js 기본 도형으로 만드는 것

| 요소 | Three.js 구현 방식 |
|---|---|
| 바닥 | PlaneGeometry + 전통 질감 텍스처 |
| 진입 동선/길 | 얇은 BoxGeometry 또는 PlaneGeometry |
| 별자리 하늘 | SphereGeometry(반전) + 별 텍스처 또는 Shader |
| 파티클 별 효과 | Points + BufferGeometry + 커스텀 셰이더 |
| AI 콘텐츠 게시판 | PlaneGeometry 액자틀 + 이미지/영상 텍스처 |
| 게시판 테두리 | 전통 문양 텍스처로 동양풍 액자 스타일 |
| QR 존 | PlaneGeometry + QR 이미지 텍스처 |

# 3. 3D 공간 구성 (최종)

```
[월정교 교각] → 입장 동선, 다리를 건너는 느낌
       ↓
[별자리 하늘 + 파티클 효과] → 천문대 분위기, 신라의 밤하늘
       ↓
[첨성대 메인 오브젝트] → 클릭 시 사이드 패널에 설명 + AI 창작 UI 표시
       ↓
[병풍형 AI 콘텐츠 게시판] → PlaneGeometry 액자들에 창작물 전시
       ↓
[QR 오프라인 연결 패널] → 클릭 시 QR 표시 → 청중이 스마트폰으로 스캔
```

# 4. UI 레이아웃: 3D + 사이드 패널 하이브리드

3D 공간에 "패널 오브젝트"는 시각적으로 보여주되, 실제 인터랙션은 화면 오른쪽 고정 사이드 패널(HTML)에서 처리한다.

```
┌──────────────────────────────┬─────────────────────┐
│                              │                     │
│     3D 공간 (70%)            │  사이드 패널 (30%)   │
│                              │                     │
│  첨성대 + 별 + 월정교         │  [상태에 따라 변경]   │
│                              │                     │
│  오브젝트 클릭 →             │  설명 / AI 창작 /    │
│  사이드 패널 내용 변경        │  결과물 / QR 등      │
│                              │                     │
└──────────────────────────────┴─────────────────────┘
```

### 사이드 패널 상태 흐름

```
[상태 1: 초기]
  → "이어줄게" 로고 + 간단한 안내 문구
  → "첨성대를 클릭해보세요"

[상태 2: 첨성대 클릭 후 — 설명]
  → "선덕여왕의 첨성대"
  → 역사적 설명 텍스트
  → [AI 콘텐츠 창작하기] 버튼

[상태 3: AI 창작 모드]
  → 스타일 선택: "별빛 국악" / "사이버 신라" / "몽환적 밤하늘"
  → 형식 선택: 이미지 / 노래 / 영상
  → [창작하기] 버튼
    → 이미지: OpenAI API 실시간 호출 (로딩 애니메이션)
    → 노래: 사전 제작물 재생
    → 영상: 사전 제작물 재생

[상태 4: 결과 표시]
  → 생성된/선택된 콘텐츠 표시
  → [게시판에 공유하기] 버튼
  → [QR로 실제 방문 이어가기] 버튼

[상태 5: QR 오프라인 연결]
  → QR 코드 표시
  → "이 QR을 스캔하면 첨성대 방문 정보를 확인할 수 있습니다"
  → 청중 스마트폰에서 우리 Vercel 페이지 오픈
```

# 5. AI 콘텐츠 생성 구조

## 실시간 생성 (이미지)

```
사용자가 스타일 + "이미지" 선택
  → 프론트엔드가 프롬프트 빌더 호출
  → 프롬프트 예시:
    "Cheomseongdae observatory under starry night sky,
     Korean traditional atmosphere, [선택한 스타일],
     digital art, high quality"
  → OpenAI API 호출 (POST /api/generate-image)
  → 로딩 중: "AI가 선덕여왕의 별을 다시 그리고 있습니다..." 애니메이션
  → 생성된 이미지 사이드 패널에 표시
  → 3D 게시판에도 자동 추가
```

## 사전 제작 (노래/영상)

```
사용자가 스타일 + "노래" 또는 "영상" 선택
  → contentMap에서 사전 제작물 경로 조회
  → 미디어 플레이어로 재생

contentMap 예시:
{
  "starlight-gukak": {
    style: "별빛 국악",
    song: "/media/cheomseongdae_starlight_gukak.mp3",
    video: "/media/cheomseongdae_starlight_short.mp4",
    images: [
      "/media/cheomseongdae_poster_01.png",
      "/media/cheomseongdae_poster_02.png"
    ]
  }
}
```

## 사전 제작물 목록

| 종류 | 파일 | 설명 |
|---|---|---|
| 노래 1곡 | cheomseongdae_starlight_gukak.mp3 | Suno AI 생성, 국악 lo-fi + 별빛 테마 |
| 영상 1개 | cheomseongdae_starlight_short.mp4 | 30초 내외, 첨성대 + 선덕여왕 숏폼 |
| 이미지 2개 | cheomseongdae_poster_01.png | 첨성대 별빛 포스터 스타일 |
|  | cheomseongdae_poster_02.png | 사이버 신라 스타일 |

# 6. AI 콘텐츠 게시판 구현

## 성격

더미 데이터를 포함한 **커뮤니티형 게시판**. 내가 만든 것 + 다른 사용자(가상)가 만든 것이 함께 표시된다.

## 3D 구현 방식

```
- PlaneGeometry 액자 5~6개를 병풍처럼 반원형 배치
- 각 액자에 이미지 텍스처 입힘
- 전통 문양 테두리 텍스처로 동양풍 액자 스타일
- 액자 클릭 시 사이드 패널에 해당 콘텐츠 상세 표시
- 노래가 포함된 카드는 클릭 시 오디오 재생
```

# 7. QR 오프라인 연결 구현

## 구현 방식

QR 코드에 우리 서비스의 Vercel 배포 URL을 인코딩한다.

```
QR 내용: https://ieojulge.vercel.app/offline/cheomseongdae
```

## 오프라인 페이지 구성

```
/offline/cheomseongdae 페이지 내용:

1. 첨성대 사진 + 소개
2. 위치 정보 (경주시 인왕동)
3. 국가유산 포털 링크 (heritage.go.kr)
4. 주변 문화유산 방문 코스 추천
   - 동궁과 월지 (도보 10분)
   - 대릉원 (도보 15분)
   - 월정교 (도보 20분)
5. "이어줄게에서 만든 내 AI 콘텐츠" 섹션 (로컬스토리지 기반)
```

## 발표 시 활용

```
시연 중 QR 코드가 3D 공간에 표시됨
  → 발표자: "지금 이 QR을 스캔해보시면
    첨성대 방문 정보를 확인하실 수 있습니다"
  → 청중 스마트폰에서 우리 서비스 페이지 오픈
  → 실제 작동하는 오프라인 연결 시연
```

# 10. MVP 완료 기준

해커톤 제출 기준으로 "완성"은 기능이 많다는 뜻이 아니라, 발표 시나리오가 끊기지 않고 끝까지 보이는 상태를 의미한다.

| 기능 | 완료 기준 |
|---|---|
| 랜딩 페이지 | `체험 시작하기` 버튼으로 3D 체험 페이지 진입 |
| 3D 공간 | 브라우저에서 첨성대, 별자리 효과, 입장 동선이 정상 표시 |
| 첨성대 클릭 | 클릭 시 사이드 패널이 설명 상태로 전환 |
| AI 창작 UI | 스타일과 형식 선택 후 결과 표시 흐름 동작 |
| 실시간 이미지 생성 | OpenAI API 1회 이상 실제 호출 성공 |
| 노래/영상 | 사전 제작 파일이 5~8초 이상 안정적으로 재생 |
| 게시판 | 더미 콘텐츠 + 생성 결과가 카드/액자 형태로 표시 |
| QR 연결 | QR 또는 버튼으로 `/offline/cheomseongdae` 페이지 접속 |
| 모바일 확인 | QR 타겟 페이지가 스마트폰 화면에서 깨지지 않음 |

## 발표 데모 최소 성공선

```
랜딩 → 3D 공간 → 첨성대 클릭 → AI 이미지 결과 → 게시판 → QR 페이지
```

이 흐름만 안정적으로 되면 MVP는 성공이다.

# 11. 시연 실패 대응

## OpenAI API 실패

```
문제:
  - 네트워크 지연
  - API 키 문제
  - 호출 제한

대응:
  - 사전 생성 이미지 1~2장을 fallback으로 준비
  - 실패 시 "네트워크 상황에 따라 사전 생성 결과를 보여드리겠습니다" 문구 표시
  - 발표자는 "이미지 생성은 API 구조까지 연결했고, 현재는 백업 결과로 시연합니다"라고 말함
```

## 3D 모델 로딩 실패

```
문제:
  - 첨성대 모델 용량이 큼
  - 텍스처 로딩 지연

대응:
  - 경량화된 GLB 또는 텍스처 제거 버전 준비
  - 로딩 화면을 5초 이상 보여주지 않도록 Suspense fallback 구성
  - 최악의 경우 첨성대 대체용 기본 도형/이미지 패널 표시
```

## 미디어 재생 실패

```
문제:
  - 브라우저 자동재생 제한
  - 파일 경로 오류

대응:
  - 반드시 버튼 클릭 후 재생
  - 영상/노래 썸네일과 설명 텍스트를 함께 준비
```

## QR 스캔 실패

```
문제:
  - 카메라 인식 실패
  - 배포 URL 접속 지연

대응:
  - QR 아래에 짧은 URL도 함께 표시
  - 발표자 노트에 직접 열 수 있는 URL 준비
```

## 별자리 파티클 효과

```tsx
import { useRef, useMemo } from "react";
import { useFrame } from "@react-three/fiber";
import * as THREE from "three";

function StarField({ count = 2000 }) {
  const points = useRef();
  const positions = useMemo(() => {
    const pos = new Float32Array(count * 3);
    for (let i = 0; i < count; i++) {
      pos[i * 3] = (Math.random() - 0.5) * 100;
      pos[i * 3 + 1] = Math.random() * 50 + 10;
      pos[i * 3 + 2] = (Math.random() - 0.5) * 100;
    }
    return pos;
  }, [count]);

  useFrame((state) => {
    points.current.rotation.y = state.clock.elapsedTime * 0.02;
  });

  return (
    <points ref={points}>
      <bufferGeometry>
        <bufferAttribute
          attach="attributes-position"
          count={count}
          array={positions}
          itemSize={3}
        />
      </bufferGeometry>
      <pointsMaterial size={0.15} color="#ffeedd" sizeAttenuation />
    </points>
  );
}
```

## 3D 오브젝트 클릭 → 사이드 패널 연동

```tsx
// useAppStore.ts (Zustand)
import { create } from "zustand";

interface AppState {
  selectedObject: string | null;
  panelMode: "idle" | "info" | "create" | "result" | "qr";
  setSelectedObject: (id: string) => void;
  setPanelMode: (mode: AppState["panelMode"]) => void;
}

export const useAppStore = create<AppState>((set) => ({
  selectedObject: null,
  panelMode: "idle",
  setSelectedObject: (id) => set({ selectedObject: id, panelMode: "info" }),
  setPanelMode: (mode) => set({ panelMode: mode }),
}));
```

# 14. 성능 최적화 고려사항

```
1. 첨성대 모델이 1.1M triangles로 무거움
   → Blender 또는 gltf-pipeline으로 Draco 압축 적용
   → 또는 LOD(Level of Detail) 구성

2. 텍스처 최적화
   → 2048x2048 이하로 리사이즈
   → WebP 포맷 사용

3. 파티클 수 조절
   → 데스크탑 2000개, 모바일 500개

4. 에셋 Lazy Loading
   → useGLTF.preload()로 사전 로딩
   → Suspense + 로딩 화면 제공
```
