# 이어줄게 — 기타 페이지

삭제하기에는 아쉽지만 발표 본문에 넣으면 산만해질 수 있는 내용, 확장 아이디어, 출처를 정리한 페이지.

---

# 1. 대안 테마: 불국사 · 석굴암 · 김대성 설화

## 왜 별도 보관하는가

첨성대+선덕여왕을 메인 테마로 확정했지만, 불국사·석굴암의 김대성 설화는 "이어줄게"의 확장 사례로 활용할 수 있다.

발표 본문에서는 길게 말하지 않는다. 확장 가능성 슬라이드에서 한 문장 정도만 사용한다.

## 김대성 전생·현생 설화 요약

신라 시대 김대성은 전생과 현생의 부모에게 보답하기 위해 두 사찰을 세웠다고 전해진다.

- **석굴암(석불사)**: 전생의 어머니를 위한 절
- **불국사**: 현생의 부모를 위한 절

이 설화는 "과거와 현재를 잇는다"는 점에서 이어줄게의 확장 테마로 쓸 수 있다.

## PPT에서 사용할 수 있는 짧은 멘트

> "이 구조는 첨성대에만 머물지 않습니다. 불국사·석굴암처럼 고유한 이야기를 가진 문화유산에도 적용할 수 있습니다."

## 출처 상태

- 김대성 설화 출처: `출처 조사 필요`
- 발표에 넣을 경우 《삼국유사》 기반 설명 또는 공신력 있는 백과/기관 출처 확인 필요

# 4. 제품 확장 로드맵

발표에서는 전부 설명하지 않는다. 질문이 들어왔을 때 답변용으로 사용한다.

| 단계 | 내용 |
|---|---|
| MVP | 첨성대 단일 테마, 3D 체험, 이미지 실시간 생성, 노래/영상 사전 제작, QR 방문 연결 |
| v1 | 불국사·석굴암 등 경주 문화유산 테마 확장 |
| v2 | 문화유산별 AI 창작 템플릿 확장: 이미지, 음악, 숏폼 |
| v3 | 공개 3D 에셋이 없는 지역 문화유산을 3D Gaussian Splatting으로 캡처 |
| v4 | 학교·지자체·관광기관에서 쓸 수 있는 문화유산 체험/창작 플랫폼 |

---

# 5. 3D Gaussian Splatting 확장 계획

## 현재 (MVP)

```
Three.js + 공개 국가유산 GLB 에셋
```

## 확장 단계

```
공개 에셋이 없는 지역 문화유산
  → 현장 촬영
  → 3D Gaussian Splatting으로 공간 캡처
  → 웹 기반 뷰어로 체험
  → 기존 AI 콘텐츠 창작 구조에 연결
```

## 발표 멘트

> "이번 MVP는 공개 3D 에셋 기반으로 구현했습니다. 공개 에셋이 없는 문화유산은 추후 3D Gaussian Splatting으로 확장할 수 있습니다."

---

# 6. RAG 확장 계획

## 현재 (MVP)

```
출처 기반 문화유산 설명 데이터를 직접 작성
```

## 확장 단계

```
국가유산 자료, 이미지, 도면, 설명문 수집
  → 멀티모달 RAG로 인덱싱
  → 사용자 질문에 답변하는 AI 문화해설사
```

## 발표 사용 여부

이번 발표 본문에는 넣지 않는다. 질문이 나오면 "확장 가능성"으로만 짧게 답한다.

# 8. 삭제된 기존 기획 요소

## "순환" 용어

- 기존: "전통문화 순환 플랫폼"
- 변경: "전통문화 이어짐 플랫폼"
- 이유: 온라인→AI→오프라인은 순환보다 연결/이어짐에 가깝다. 기획명 "이어줄게"와 일관성이 높다.

## "리믹스" 용어

- 기존: "AI 리믹스", "리믹스 스튜디오", "리믹스 피드 월"
- 변경: "AI 콘텐츠 창작", "창작 공간", "콘텐츠 게시판"
- 이유: 발표자가 설명하기 쉽고, 이미지·노래·영상을 모두 포괄할 수 있다.

## 유저 간 재리믹스 기능

- 기존: 다른 사용자의 결과물을 다시 리믹스하는 기능
- 변경: 삭제. 게시판에서 보고 공유하는 선에서 종료.
- 이유: MVP 범위가 커지고, "단절을 잇는다"는 핵심과 거리가 생긴다.

## 5개 구역 분리

- 기존: 입장 게이트 / 중심 공간 / 스튜디오 / 피드 월 / 오프라인 존
- 변경: 하나의 통합 3D 공간 + 사이드 패널
- 이유: 시연 시간 3분 안에 설명하기 어렵다. 하나의 공간에서 자연스럽게 흐르는 구성이 낫다.

## 정림사지 테마

- 기존: 정림사지 / 백제 모티프
- 변경: 첨성대 / 경주 신라 테마
- 이유: 에셋, 대중 인지도, 선덕여왕 스토리텔링, "이어줄게" 내러티브 연결이 더 좋다.

## Heritage Loop 기획명

- 기존: Heritage Loop
- 변경: 이어줄게
- 이유: 문제(단절)와 해결(이어짐)을 한국어 이름 하나로 전달할 수 있다.
